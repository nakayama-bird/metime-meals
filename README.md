# **MeTime Meals**（ミータイムミールズ）（仮）

## ■サービスの概要

「MeTime Meals（ミータイムミールズ）（仮）」は、「お一人様外食」に特化した飲食店投稿アプリです。一人だからこそ快適な空間で、周りを気にすることなく食事を楽しめる飲食店を探したいという方々が情報を共有することで、ソロ活の強い味方になります。地図上や個性的なタグからご自身のニーズにあったお店を探すことができます。

## ■このサービスへの思い・作りたい理由

前職が当直のシフト勤務であったため、友人と休みが合わず一人で外食する機会が多々ありました。その際、食べログなどのレビューサイトやGoogleマップのレビューを参考にしながらお店を探すのですが、一人で入店しても大丈夫なお店なのかを調べるのに苦労しました。そこで、同じように一人で外食している方の生の声を集めることで、お一人様外食のお店探しの一助になればいいと思いこのサービスの開発を決意しました。

## ■このサービスの開発にあたっての事前調査
今回のアプリを開発するにあたり一人外食のお店選びの実情及び実装する機能に関して、調査システム：Questantを利用してアンケートを実施しました（回答数:110）。以下こちらのアンケートについては「事前アンケート（設問番号）」と表記します。

## ■ユーザー層について

- 一人で外食する20~30代女性
- 事前アンケート（Q2）で、一人で外食する際、飲食店レビューサイト（29.7%）やInstagramやXのハッシュタグ（#一人飯、ソロ飯）（28.4%）で検索していると回答した層に向けたアプリケーションをイメージ

## ■サービス利用のイメージ
#### 一人外食に特化したお店の投稿
- 一人で外食した時のみ投稿できるサービスにしたいと考えています。
- その理由として、事前アンケート（Q3）で、一人外食の検索ツールの課題として、29.7%の方が「一人向けの情報が少ない」と回答していることや、私個人の実感としても「実際に一人で外食した方の意見」を探すことに苦労した経験があるためです。
- 投稿ページに「このアプリはお一人様外食専用のアプリです。こちらの投稿は一人で外食した際の内容でお間違いないですか？」と言ったチェックボックスを設けることや投稿内容の見回りをすることで、一人外食の投稿であることを確認していきます。

#### Google Maps APIを活用した機能
- 投稿の際、Google Maps APIを利用してお店の場所の情報を保存できるようにします。
- Google Maps JavaScript APIを利用して、投稿した場所を地図上にピンで表示できるようにします。またGeolocation APIを利用して現在地周辺のお店を検索できる機能を実装します。この2つの機能に関しては、事前アンケート（Q6）でも必要な機能として上位にあったため優先的に実装を進めます。

#### 一人外食のお店選びで参考になりそうなタグの設定
- 「おひとり様用のメニューがある」「カウンター席がある」「注文をタブレットなどでできる」など一人で外食する際、ポイントになりそうなものを簡単にタグ付けできるようにします。
- 店内の雰囲気（空間の明るさ、広さ、騒がしさ）や料理の特徴（お通しが良い、ドリンクにセンスあり）もタグに含めることで、個人のニーズに合わせたお店選びをできるようにしていきます。
- これらの項目を検索できるようにして、簡単にお店選びをできるようにします。

#### いいねボタン
- いいねボタンを実装することで、気になるお店の投稿をいいねできます。またマイページにいいねした投稿の一覧を作成することで、気になるお店を見返すことができます。

#### 投稿数に応じたバッチの付与
- 投稿数に応じてバッチを付与することで、投稿へのモチベーションにつなげます。投稿件数やエリア、ジャンルなどを評価し、バッチを付与できる機能を実装します。

#### 飲食した際の金額の入力（任意項目）
- 飲食に使った金額をおおまかに入力できるようにしようと考えています。
- 事前アンケート(Q5)で、一人外食をする際お店選びで重要視する点として、51.4%の方が価格が手頃という回答をしており、おおよその予算感が分かるのが重要だと考えたからです。

#### ユーザーの性別・年代の入力（任意項目）
- 投稿した人の性別や年代がわかることで、どのような人がお店に行っているかのイメージがつきやすくなると考えました。

## ■ユーザーの獲得について
#### Xシェア機能から
- 各投稿からXシェアできるようにすることでユーザー拡大を目指します。
#### 検索エンジンからのアクセス
- meta-tagやパンくずの設定を行うことでSEO対策を行い、検索エンジン最適化を図ります。

## ■サービスの差別化ポイント・推しポイント
#### 【比較したサービス①】飲食店レビューサイト（[食べログ](https://tabelog.com/)/[ぐるなび](https://www.gnavi.co.jp/)/[Retty](https://retty.me/)）
【差別化ポイント】
- 「一人」などのタグで検索して探すことはできるものの、一人用のお店に特化しているわけではないので、必ずしも正しい情報が集められるわけではありません。
- 例えば食べログで、[【お一人様OK！】東京でおすすめの居酒屋をご紹介！](https://tabelog.com/tokyo/rstLst/cond04-00-02/izakaya/)を見ると、「一人3品頼めて60分飲み放題」や「串は一人一本にしました」といったように一人で行ったのではない情報も混在しているため探しにくいです。

=>このアプリでは「お一人様外食」に特化しているため、必ず一人で外食したところから投稿を探すことができます。
#### 【比較したサービス②】 [ひとり外食専用アプリ「ソロメシ」](https://www.solomeshi.jp/#sec-qa)
【差別化ポイント】
- 一人で確実に入りやすい店を紹介していますが、エリアと店舗が限定されており使えるユーザーが限定的です。

=>このアプリではエリアは日本全国の「お一人様外食」の投稿を想定しており、さらに幅広い層が利用可能になると考えます。
